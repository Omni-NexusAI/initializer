## Fixed Custom Skills Not Showing in UI

**Problem:** Custom skills (simplemem, initializer, agent-voice-messaging) weren't appearing in control UI or slash commands.

**Root Cause:** Missing YAML frontmatter in SKILL.md files. OpenClaw requires frontmatter with `name`, `description`, and `metadata` fields.

**Solution:** Added frontmatter to each skill:
```yaml
---
name: skill-name
description: Description here
metadata:
  { "openclaw": { "emoji": "🎙️", "requires": { "bins": ["mcporter"] } } }
---
```

**Result:** All three skills now show as "Ready" in `openclaw skills list` and `openclaw skills check`.

**Lesson:** SKILL.md must have YAML frontmatter for UI integration. Plain markdown title isn't enough.

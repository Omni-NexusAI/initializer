## User Questions/Requests
- **Voice skill extensibility**: User asked if the skill can handle special features from different TTS/STT models (e.g., Cartesia voice design, emotional prompting, voice blending) without being explicitly built for them
- **Answer**: Yes - modular architecture with provider factory pattern. Each provider accepts kwargs from config, so new features can be added by extending provider classes.
- **Auto-processing**: User asked if installing the skill on another agent will automatically handle voice messages. **Answer: No** - the skill is a library that provides `transcribe()` and `synthesize()` methods, but the automatic trigger when a voice message arrives is NOT built into the skill. The `auto_process_voice = true` config is just a value with no code that reads it. The agent's message handling layer needs to call the skill when voice messages arrive.
